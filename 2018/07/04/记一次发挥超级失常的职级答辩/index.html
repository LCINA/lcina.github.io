记一次职级答辩...

发表于 2018-07-04 | 阅读次数:

评委问的问题回答比较糟糕。后面一些问题直接就不想回答了，这个是个很严重的错误。其实一些问题在项目中确实很多地方用到，但是只要被问到一些实现机制、一些替代方法可能就直接pass了。所以前端不要局限于单纯解决问题，关键还是要想想我用这个方法，它是如何去实现的，如果让自己去解决这个问题，不依赖于库或者插件，自己怎么去实现。不逼逼了，列下一些还记得的问题做做记录。

  1. flux还是redux
  2. webpack打包是一个怎样的过程,babel的作用；
  3. 项目中遇到问题如何定位，如何解决;
  4. 题目：async function a { return 1; }，执行a()，返回什么；
  5. js有哪些继承方式，列举一下；
  6. commonjs、amd、cmd、webpack的一些规范；
  7. http返回码，前端请求接口用了get，而后端用的是post，返回码是什么。返回码422表示什么；
  8. promise的其他api，除了then、catch这些；
  9. 在不用promise和async这些异步操作处理情况下，自己如何解决遇到的异步问题，多个异步调用怎么做；
  10. 一些路由库实现了修改url，改变了界面，但是浏览器不触发刷新，如果自己做，你会怎么解决；
  11. 说一下盒模型；
  12. 说一下git的一些操作；
  13. 如何实现一个B类继承A类的某些方法；
  14. js静态类；
  15. 文档流，脱离文档流的操作；
  16. Object.assign是深拷贝还是浅拷贝；

  17. 设计模式——观察者模式
